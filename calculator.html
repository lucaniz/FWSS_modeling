Methodology — how the models work click to expand
provePossession model
Gas scales logarithmically with piece count N (number of Merkle roots in the dataset). This is because provePossession traverses a Sum Tree of depth log₂(N) on-chain — 5 challenges × O(log₂N) SLOAD operations. Piece size (leafCount) is irrelevant — it's stored as a value in Sum Tree nodes for proportional sampling, not as tree depth.
gas = 158.67M + 8.485M × log₂(pieces)
R² = 0.9551 · MAE = 4.8M gas · fitted on 670 real mainnet datasets · refitted daily
removePieces model
Gas scales sublinearly with K (pieces removed per tx) because within the same transaction, multiple pieces share Sum Tree ancestor nodes — warm access costs ~200 gas instead of ~5,000. The O(K × log₂N) theoretical formula from the contract code is not clearly confirmed by observed data (N is not tracked at removal time in FOC Observer). Both calculators use the same model: observed values for K≤144, linear interpolation between observed points, K^0.614 extrapolation beyond K=144 (unobserved).
Gas fee model
From Lotus source: effective_gas_price = BaseFee + min(GasPremium, GasFeeCap − BaseFee)
Total cost = GasUsed × BaseFee (burned) + GasUsed × effective_premium (to SP) + overestimation burn.
Today: BaseFee ≈ 100 attoFIL, GasPremium ≈ 252,000 → SP tip dominates (99.9% of fee). Under FIP-0115 (Draft): BaseFee would rise to ~90% of EGP.
